# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Running the site

No build step. Open `index.html` directly in a browser:

| Platform | Command |
|---|---|
| macOS | `open index.html` |
| Linux | `xdg-open index.html` |
| Windows | `start index.html` |

## Architecture

Everything lives in a single `index.html` â€” no external dependencies, no framework, no build tool. All CSS is in one `<style>` block in `<head>`; all JS is in one `<script>` block before `</body>`. The PDF resume, the logo files, and `apple-touch-icon.png` live in `assets/`.

**Logo & icons** â€” the brand mark is a monoline `<mf/>` glyph (purple `#8c6edc`, `viewBox="0 0 47 24"`). The **single source of truth is `assets/logo.svg`** (transparent, reusable anywhere). Derived assets: `assets/logo.png` (1024-wide transparent raster), `assets/logo-square.png` (512Â˛ transparent), `assets/icon-square.svg` (glyph centered on a dark `#1e1e1e` square), and `assets/apple-touch-icon.png` (180Ă—180 **opaque** PNG rendered from `icon-square.svg`; iOS rounds the corners). The **browser-tab favicon** is the same glyph inlined as an SVG data URI on the `<link rel="icon">` (so `#` is `%23`-encoded and attrs use single quotes). iOS ignores SVG favicons for "Add to Home Screen", so `<link rel="apple-touch-icon" href="assets/apple-touch-icon.png">` supplies the home-screen tile. To regenerate the PNGs after editing `logo.svg`, render via Playwright/WebKit (`element.screenshot`, `omit_background=True` for transparent ones) and flatten the apple-touch-icon to RGB (no alpha). Changing the glyph means updating `logo.svg`, the inline favicon data URI, and re-rendering the PNGs.

**Accent color** â€” the brand accent is purple `#8c6edc` (RGB `140, 110, 220`), exposed as `--accent`. Many effects (glow shadows, canvas particles, the promo badge) use the raw `rgba(140, 110, 220, â€¦)` triplet inline rather than the variable, so changing the accent means updating both the `--accent` declarations and these literal values.

**Theming** â€” light/dark mode uses CSS custom properties on `:root`, with a `[data-theme="dark"]` override block. Setting `data-theme="dark"` on `<html>` activates the dark palette; removing it restores light. **The site defaults to dark** â€” the inline `<head>` script applies `data-theme="dark"` **before first paint** (unless `localStorage.theme === 'light'`) to avoid a light-background flash on load; the script at the end of `<body>` re-applies it and wires the toggle. `html` also has `background: var(--bg)` so the root surface is themed from frame one. The toggle's checked state uses `label:has(input:checked) .toggle-track`; keyboard focus uses `label:has(input:focus-visible) .toggle-track`.

**Scroll position** â€” the head script sets `history.scrollRestoration = 'manual'` so reloads always start at the top (otherwise the browser restores the prior scroll position, which breaks the intro/hero reveal).

**Header & nav** â€” a single sticky bar (`position: sticky; top: 0`) containing the hamburger button, `<nav class="site-nav">`, and the theme toggle. There is no name/brand in the header. **Glassmorphism blur lives on `header::before`, not on `header` itself** â€” if `<header>` carried `backdrop-filter` it would become a "backdrop root" and the mobile nav dropdown (a descendant) could no longer blur the page behind it. Nav links are `href="#section-id"` anchors, but clicks call `e.preventDefault()` + `scrollIntoView({ behavior: 'smooth' })` so no hash is added to the URL. **iOS ghost-underline fix** â€” on iOS Safari the header rides a `backdrop-filter` compositing layer that is **not** repainted when only a class changes, so each previously-active tab's `border-bottom` stays painted and several tabs appear highlighted at once (the DOM still has exactly one `.active`; it is purely a paint/invalidation bug, and desktop WebKit does not reproduce it). `setActiveLink` therefore (a) early-returns when the active id is unchanged and (b) on a real change toggles `transform: translateZ(0)` on `.header-inner` for one frame to force a recomposite that clears the ghosts. Do not also layer-promote `.nav-link` (e.g. `transform: translateZ(0)`) â€” that would move the links onto their own layers and the `.header-inner` nudge would no longer repaint them.

The hover-brighten on nav links is written as `html:not(.touch) .nav-link:hover` â€” **not** `@media (hover: hover)`, because iPadOS Safari matches `hover: hover` (it reports as desktop-class) and would still leave `:hover` "stuck" on every tapped tab, showing several phantom highlights at once next to the real `.active` one. JS adds `html.touch` on the first `touchstart`, which reliably disables the hover rule on touch devices. (Other hover effects â€” banners, project cards, buttons â€” are intentionally left un-gated; their lift/glow reads as acceptable tap feedback.) A scroll-position scrollspy (rAF-throttled `scroll`/`resize` listener, `updateActiveLink`) adds `active` to the link of the **last section whose top has scrolled above a reference line** (`ACTIVE_OFFSET = 100px`, just below the header). This replaced an earlier `IntersectionObserver` detection band that couldn't highlight a short last section. **Bottom-of-page exception:** the trailing sections (Projects + Contact + footer) are together shorter than a tall (tablet) viewport, so when the page is pinned at the bottom they share the same scroll position and can't reach the reference line â€” geometry alone can't tell them apart. So `updateActiveLink` special-cases `atPageBottom()`: it honors `navIntentId` (the id of the last nav link the user clicked) there, falling back to the last section. A real user scroll (`wheel`/`touchmove`/`keydown`, which a programmatic smooth scroll does **not** fire) clears `navIntentId`, so manually scrolling to the bottom resolves to Contact normally. The nav click handler sets `navIntentId` and calls `requestActiveUpdate()` (needed because clicking between Projects/Contact while already pinned at the bottom produces no scroll event). Sections use `scroll-margin-top: 72px` for the 56px-tall bar. On mobile the dropdown animates open/closed via `opacity` + `transform` + `visibility` (not `display`, so it can transition and so its blur renders).

**Hero (`#home`)** â€” contains a `<canvas class="home-canvas">` particle network. The particle code is a reusable `createParticleField(canvas, opts)` factory returning `{ start, stop }`, shared by both the hero canvas and the intro overlay. The hero field's rAF loop is gated by an `IntersectionObserver` on the canvas: it pauses when the hero scrolls off-screen and resumes when it returns (saves the main thread / battery). The tagline runs a **typing animation** that waits for the `intro:done` event (so it types out as the page is revealed, not while hidden under the intro). The hero name, tagline, and banners reveal via the intro's staggered `.intro-rise` animation (see **Intro reveal**).

**Intro reveal** â€” on a genuine first visit per session, the head script adds the `intro` class to `<html>` (gated by `sessionStorage.introSeen`; skipped for `prefers-reduced-motion` and no-JS). A full-viewport `#intro` overlay runs its own particle field. After ~1.3s (or on first click/scroll/key/touch), `finish()` swaps `intro` â†’ `intro-done` on `<html>` (triggering the staggered `@keyframes introRise` on `.intro-rise` elements) and animates the overlay **converging into the hero canvas's bounding box** (translate + scale) while fading, so the field gathers into place rather than being replaced. The overlay keeps an `intro-closing` class so it stays displayed (its base styles aren't gated behind `html.intro`) through the exit animation, then is removed.

**Banners** â€” three `<a class="banner">` cards (LinkedIn, GitHub, CV) in a `div.banners` flex row. Icons are inline SVGs. The LinkedIn URL is hardcoded in the `href`. **GitHub and CV are both `banner--soon` placeholders** ("Coming soon"): non-interactive via `pointer-events: none`, `tabindex="-1"`, `aria-disabled="true"`, `href="#"`. To activate one, restore its real URL and remove the `banner--soon` class plus those attributes. Banners share the purple lift+glow hover with project cards.

**Experience** â€” a vertical **timeline**: `.experience-list` draws the line (`::before`), each `.exp-entry` draws its dot (`::before`). A `dotObserver` IntersectionObserver toggles `.dot-active` on the in-view entry so only its dot pulses. The promotion line (`.exp-promo`) renders a small purple circular `â†‘` badge via `::before`.

**Skills** â€” 2-column grid of `.skill-category` cards, each with a `--bar-width` (set inline) and a `--cat-rgb` triplet (currently unified to purple on the base rule). The animated fill bar fills via a `width` transition triggered when the card gets `is-visible` from the scroll-animation observer. Each card shows a `.skill-level` label; tags are tinted with `rgba(var(--cat-rgb), â€¦)`.

**Skill popover** â€” hovering (desktop) / tapping (mobile) a `.skill-tag` shows a shared `.skill-popover` element (one node appended to `<body>`, repositioned per tag, flips above/below based on room) with a fade-and-scale animation. Per-skill copy lives in the editable `skillInfo` object keyed by exact tag text; tags without an entry show a fallback. A `lastShow` timestamp guard prevents the synthesized focus/click on touch from instantly dismissing the popover.

**Projects** â€” `.project-card` grid; cards are flex columns so the GitHub link (`.project-link`, pinned with `margin-top: auto`) sits at the bottom regardless of text length. The links are "Coming soon" placeholders (`href="#"`, `tabindex="-1"`, `aria-disabled="true"`); swap in the real repo URL and remove those attributes when ready. Hover gives a purple border, lift, glow, and accent title.

**Contact** â€” a single two-column `.contact-card`: heading + tagline on the left, a purple "Send a message" `mailto:` CTA button plus website/location icon rows on the right.

**Scroll animations** â€” an inline script in `<head>` adds `first-visit` to `<html>` before CSS loads (prevents FOUC). `.exp-entry` and `.skill-category` start hidden (`opacity: 0; translateY`) and get `is-visible` from an `animObserver` IntersectionObserver. (The hero name is no longer in this set â€” it's handled by the intro `.intro-rise` reveal.) Animations play on every load (no sessionStorage gate), except under `prefers-reduced-motion: reduce` â€” a media block forces the reveal elements (`.exp-entry`, `.skill-category`, `.project-card`, `.contact-card`) to their final state with no slide/fade.

**Sections** â€” six `<section id="...">` inside `<main>`: `home`, `about`, `experience`, `skills`, `projects`, `contact`. Each has class `section`; adjacent sections separated by `border-top`. `#skills`, `#projects`, `#contact`, and `footer` use `content-visibility: auto` (with `contain-intrinsic-size`) so off-screen sections skip layout/paint â€” this cuts the reflow cost when the experience list expands/collapses on mobile.

**Mobile collapse** â€” inside the JS, a `matchMedia('(max-width: 520px)')`-driven IIFE makes long text expandable and is **responsive to resize**: `setup()` applies the collapse under 520px, `teardown()` removes the injected buttons and strips the inline styles above 520px (so resizing to desktop restores the full text), driven by the `matchMedia` `change` listener (`data-collapsible` guards prevent double setup). The About paragraph collapses to ~80px with a fade-out mask + "Read more"/"Read less"; each experience bullet list with >2 items collapses by animating the `<ul>`'s own `height` (single element = one layout recalc per frame), with the hidden `<li>`s fading via staggered `opacity`. Buttons are injected by JS only on small screens â€” no duplicate content.

**Responsive** â€” the `@media (max-width: 520px)` rule shows the hamburger and turns `.site-nav` into an animated, blurred dropdown, stacks banners, collapses the experience grid and date, and switches the skills grid and contact card to one column.

**Scrollbar** â€” a thin (4px) purple scrollbar via `::-webkit-scrollbar` (WebKit) and `scrollbar-color` (Firefox), with a transparent track. `.site-nav` hides its own (horizontal) scrollbar with `scrollbar-width: none` + `::-webkit-scrollbar { display: none }`.
