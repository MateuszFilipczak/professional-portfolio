# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Running the site

No build step. Open `index.html` directly in a browser:

| Platform | Command |
|---|---|
| macOS | `open index.html` |
| Linux | `xdg-open index.html` |
| Windows | `start index.html` |

## Architecture

No framework, no build tool. Two pages — `index.html` (the single-page portfolio) and `certificates.html` — share one **external stylesheet `assets/styles.css`** (moved out of the old inline `<style>` so both pages stay consistent; this is the one place to edit CSS). Each page keeps its own inline `<script>` (home logic vs. the small certificates renderer) but both load the shared **`assets/particles.js`** (the particle-network factory) via `<script src>`, and each keeps a tiny **inline pre-paint theme script** in `<head>` (applies dark default before first paint — must stay inline, can't be external). The PDF resume, `logo.svg`, and the PNG icons (`apple-touch-icon.png`, `icon-192.png`, `icon-512.png`) live in `assets/`.

**Certificates page (`certificates.html`)** — linked from a 4th home `.banner` ("Certificates", active, award-icon, `#certificates-banner`; gets the `:nth-child(4)` intro-rise delay in `styles.css`). Header (the `<mf/>` `assets/logo.svg` as a `.brand-logo` link back home, left; a `.cert-nav` with Certifications/Courses links, right — see **Header & nav**), a full-page particle background (`.page-bg-canvas`, fixed, started via the shared `createParticleField`; gated by `prefers-reduced-motion`; content sits above it via `.has-page-bg main { z-index: 1 }`), and **two groups**: **Certifications** (from `const CERTS = [{ name, company, issued, expires, status, url, logo }]`) and **Courses** (from `const COURSES = [{ name, company, completed, url, logo }]` — course completions, no status). A shared `buildCard({ name, company, dateText, status, url, logo })` renders both; `renderGroup(sectionId, listId, items, mapItem)` hides a whole `.cert-section` when its array is empty (so Courses is hidden until populated). Cert date = `Issued … · Expires …` + an `.cert-status` pill (`--active` green / `--expired` red, brighter dark variants); course date = `Completed …` with **no** pill (`status: null`). Cards are built on the `.banner` look (`.cert-card`): an `<a>` when `url` is set (hover lift), else non-clickable `.cert-card--nolink` (kept readable, unlike `.banner--soon`); `<img class="cert-logo">` (white rounded chip; logos in `assets/certificate_logo/`) when `logo` is set, else a placeholder badge SVG. Cards reveal one-by-one via the shared `@keyframes riseIn` with a per-card `animationDelay` (skipped under reduced-motion). Add entries by editing `CERTS` / `COURSES`.

**Logo & icons** — the brand mark is the literal text `<mf/>` set in the hero heading font (`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`, `font-weight: 700`, `letter-spacing: -0.03em`) in purple `#8c6edc` — it matches the `.home-name` "Mateusz Filipczak" type. Because the glyph is system-font text, it renders in San Francisco/Segoe UI/Roboto per platform (matching the site's own heading).

**`assets/logo.svg` is the only logo file kept** — transparent, a single SVG `<text>` element with a tight measured `viewBox`. It is the master, reuse it anywhere (incl. docs):
```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="15.9 19.9 351.8 169.6">
  <title>mf — Mateusz Filipczak</title>
  <text x="30" y="150" font-family="-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif" font-weight="700" letter-spacing="-0.03em" font-size="120" fill="#8c6edc">&lt;mf/&gt;</text>
</svg>
```
It is referenced as the **browser-tab favicon** (`<link rel="icon" type="image/svg+xml" href="assets/logo.svg">`) and as the **centered `.intro-logo`** in the intro overlay. The `viewBox` came from rendering the `<text>` and reading `getBBox()` (x≈30 y≈34 w≈323 h≈141) + ~10% padding; redo that if the text/font changes.

The other three asset files are **PNG icons derived from the same glyph on a dark `#1e1e1e` square** (opaque RGB, no alpha): `apple-touch-icon.png` (180², iOS "Add to Home Screen" — iOS ignores SVG favicons so this is required and rounds its own corners) and `icon-192.png` / `icon-512.png` (square contexts — Brave/Chrome new-tab tiles, PWA/Android — declared via `<link rel="icon" sizes=…>` + `site.webmanifest` at repo root; without square PNGs browsers stretch the 2:1 SVG into a square tile and it looks squeezed). `<meta name="theme-color" content="#1e1e1e">` is also set.

**Regenerating the PNG icons** (no build-helper file is kept — render this square wrapper SVG at 180/192/512 via Playwright/WebKit `element.screenshot`, then flatten to opaque RGB):
```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 100 100">
  <rect width="100" height="100" fill="#1e1e1e"/>
  <svg x="12" y="31.68" width="76" height="36.64" viewBox="15.9 19.9 351.8 169.6"><text x="30" y="150" font-family="-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif" font-weight="700" letter-spacing="-0.03em" font-size="120" fill="#8c6edc">&lt;mf/&gt;</text></svg>
</svg>
```
To export a **transparent PNG** of the logo for docs, render `logo.svg` with `omit_background=True`. Changing the mark means updating `logo.svg` and re-rendering these PNGs.

**Accent color** — the brand accent is purple `#8c6edc` (RGB `140, 110, 220`), exposed as `--accent`. Many effects (glow shadows, canvas particles, the promo badge) use the raw `rgba(140, 110, 220, …)` triplet inline rather than the variable, so changing the accent means updating both the `--accent` declarations and these literal values.

**Theming** — **dark-only.** `data-theme="dark"` is hard-coded on the `<html>` element of both pages (so the dark palette applies from frame one, no flash, no JS). The CSS still defines a light `:root` palette + a `[data-theme="dark"]` override block, but the light vars are now effectively dead (there is **no theme toggle** — it was removed). `html` also has `background: var(--bg)`. (The unused `--toggle-*` vars / light palette are kept but inert; reintroducing a toggle would mean re-adding the control + JS.)

**Scroll position** — the head script sets `history.scrollRestoration = 'manual'` so reloads always start at the top (otherwise the browser restores the prior scroll position, which breaks the intro/hero reveal).

**Header & nav** — a single sticky bar (`position: sticky; top: 0`). Layout: the `<mf/>` **`.brand-logo` (logo) is on the left** (`margin-right: auto` pushes everything else right) and the **nav is on the right**; there is no theme toggle. On the home page the logo is `href="#home"` (JS intercepts it to smooth-scroll to top), the right side is `<nav class="site-nav">` (6 links) plus the hamburger `.nav-toggle` (shown ≤520px → animated dropdown). The certificates page uses the **same nav** (`<nav class="site-nav">` + hamburger `.nav-toggle` + dropdown ≤520px) — its logo is `href="index.html"` (back home) and its two links are **Certifications / Courses** (smooth-scroll to `#certs-section` / `#courses-section`, with a scrollspy that toggles `.active`, applies the same one-frame `.header-inner` transform nudge as the home page to avoid the iOS ghost-underline bug, and hides the Courses link when that section is empty). **Glassmorphism blur lives on `header::before`, not on `header` itself** — if `<header>` carried `backdrop-filter` it would become a "backdrop root" and the mobile nav dropdown (a descendant) could no longer blur the page behind it. Nav links are `href="#section-id"` anchors; clicks call `e.preventDefault()` + `smoothScrollTo(targetEl)` (no hash added to the URL). `smoothScrollTo` uses the browser's **native** `window.scrollTo({ top, behavior: 'smooth' })` (`top = el.getBoundingClientRect().top + scrollY - 72`) — native scroll starts immediately with no JS ramp-up frame or per-frame layout reads (a custom rAF tween felt laggy, especially with the `content-visibility: auto` sections forcing per-frame layout). The scrollspy still sweeps the `.active` highlight through each section as the page scrolls. **Nav link styling:** hover (`html:not(.touch) .nav-link:hover`) and the active link both turn purple (`color: var(--accent)`); active additionally gets the accent `border-bottom`. **iOS ghost-underline fix** — on iOS Safari the header rides a `backdrop-filter` compositing layer that is **not** repainted when only a class changes, so each previously-active tab's `border-bottom` stays painted and several tabs appear highlighted at once (the DOM still has exactly one `.active`; it is purely a paint/invalidation bug, and desktop WebKit does not reproduce it). `setActiveLink` therefore (a) early-returns when the active id is unchanged and (b) on a real change toggles `transform: translateZ(0)` on `.header-inner` for one frame to force a recomposite that clears the ghosts. Do not also layer-promote `.nav-link` (e.g. `transform: translateZ(0)`) — that would move the links onto their own layers and the `.header-inner` nudge would no longer repaint them.

The hover-brighten on nav links is written as `html:not(.touch) .nav-link:hover` — **not** `@media (hover: hover)`, because iPadOS Safari matches `hover: hover` (it reports as desktop-class) and would still leave `:hover` "stuck" on every tapped tab, showing several phantom highlights at once next to the real `.active` one. JS adds `html.touch` on the first `touchstart`, which reliably disables the hover rule on touch devices. (Other hover effects — banners, project cards, buttons — are intentionally left un-gated; their lift/glow reads as acceptable tap feedback.) A scroll-position scrollspy (rAF-throttled `scroll`/`resize` listener, `updateActiveLink`) adds `active` to the link of the **last section whose top has scrolled above a reference line** (`ACTIVE_OFFSET = 100px`, just below the header). This replaced an earlier `IntersectionObserver` detection band that couldn't highlight a short last section. **Bottom-of-page exception:** the trailing sections (Projects + Contact + footer) are together shorter than a tall (tablet) viewport, so when the page is pinned at the bottom they share the same scroll position and can't reach the reference line — geometry alone can't tell them apart. So `updateActiveLink` special-cases `atPageBottom()`: it honors `navIntentId` (the id of the last nav link the user clicked) there, falling back to the last section. A real user scroll (`wheel`/`touchmove`/`keydown`, which a programmatic smooth scroll does **not** fire) clears `navIntentId`, so manually scrolling to the bottom resolves to Contact normally. The nav click handler sets `navIntentId` and calls `requestActiveUpdate()` (needed because clicking between Projects/Contact while already pinned at the bottom produces no scroll event). Sections use `scroll-margin-top: 72px` for the 56px-tall bar. On mobile the dropdown animates open/closed via `opacity` + `transform` + `visibility` (not `display`, so it can transition and so its blur renders).

**Hero (`#home`)** — contains a `<canvas class="home-canvas">` particle network. The particle code is a reusable `createParticleField(canvas, opts)` factory returning `{ start, stop }`, defined in the shared **`assets/particles.js`** (loaded via `<script src>` before each page's inline script) and used by the hero canvas, the intro overlay, and the certificates-page background. The hero field's rAF loop is gated by an `IntersectionObserver` on the canvas: it pauses when the hero scrolls off-screen and resumes when it returns (saves the main thread / battery). The tagline runs a **typing animation** that waits for the `intro:done` event (so it types out as the page is revealed, not while hidden under the intro). The hero name, tagline, and banners reveal via the intro's staggered `.intro-rise` animation (see **Intro reveal**).

**Intro reveal** — on a genuine first visit per session, the head script adds the `intro` class to `<html>` (gated by `sessionStorage.introSeen`; skipped for `prefers-reduced-motion` and no-JS). A full-viewport `#intro` overlay runs its own particle field. After ~1.3s (or on first click/scroll/key/touch), `finish()` swaps `intro` → `intro-done` on `<html>` (triggering the staggered `@keyframes introRise` on `.intro-rise` elements) and animates the overlay **converging into the hero canvas's bounding box** (translate + scale) while fading, so the field gathers into place rather than being replaced. The overlay keeps an `intro-closing` class so it stays displayed (its base styles aren't gated behind `html.intro`) through the exit animation, then is removed.

**Banners** — three `<a class="banner">` cards (LinkedIn, GitHub, CV) in a `div.banners` flex row. Icons are inline SVGs. The LinkedIn URL is hardcoded in the `href`. **GitHub and CV are both `banner--soon` placeholders** ("Coming soon"): non-interactive via `pointer-events: none`, `tabindex="-1"`, `aria-disabled="true"`, `href="#"`. To activate one, restore its real URL and remove the `banner--soon` class plus those attributes. Banners share the purple lift+glow hover with project cards.

**Experience** — a vertical **timeline**: `.experience-list` draws the line (`::before`), each `.exp-entry` draws its dot (`::before`). A `dotObserver` IntersectionObserver toggles `.dot-active` on the in-view entry so only its dot pulses. The promotion line (`.exp-promo`) renders a small purple circular `↑` badge via `::before`.

**Skills** — 2-column grid of `.skill-category` cards, each with a `--bar-width` (set inline) and a `--cat-rgb` triplet (currently unified to purple on the base rule). The animated fill bar fills via a `width` transition triggered when the card gets `is-visible` from the scroll-animation observer. Each card shows a `.skill-level` label; tags are tinted with `rgba(var(--cat-rgb), …)`.

**Skill popover** — hovering (desktop) / tapping (mobile) a `.skill-tag` shows a shared `.skill-popover` element (one node appended to `<body>`, repositioned per tag, flips above/below based on room) with a fade-and-scale animation. Per-skill copy lives in the editable `skillInfo` object keyed by exact tag text; tags without an entry show a fallback. A `lastShow` timestamp guard prevents the synthesized focus/click on touch from instantly dismissing the popover.

**Projects** — `.project-card` grid; cards are flex columns so the GitHub link (`.project-link`, pinned with `margin-top: auto`) sits at the bottom regardless of text length. The links are "Coming soon" placeholders (`href="#"`, `tabindex="-1"`, `aria-disabled="true"`); swap in the real repo URL and remove those attributes when ready. Hover gives a purple border, lift, glow, and accent title.

**Contact** — a single two-column `.contact-card`: heading + tagline on the left, a purple "Send a message" `mailto:` CTA button plus website/location icon rows on the right.

**Scroll animations** — an inline script in `<head>` adds `first-visit` to `<html>` before CSS loads (prevents FOUC). `.exp-entry` and `.skill-category` start hidden (`opacity: 0; translateY`) and get `is-visible` from an `animObserver` IntersectionObserver. (The hero name is no longer in this set — it's handled by the intro `.intro-rise` reveal.) Animations play on every load (no sessionStorage gate), except under `prefers-reduced-motion: reduce` — a media block forces the reveal elements (`.exp-entry`, `.skill-category`, `.project-card`, `.contact-card`) to their final state with no slide/fade.

**Sections** — six `<section id="...">` inside `<main>`: `home`, `about`, `experience`, `skills`, `projects`, `contact`. Each has class `section`; adjacent sections separated by `border-top`. `#skills`, `#projects`, `#contact`, and `footer` use `content-visibility: auto` (with `contain-intrinsic-size`) so off-screen sections skip layout/paint — this cuts the reflow cost when the experience list expands/collapses on mobile.

**Mobile collapse** — inside the JS, a `matchMedia('(max-width: 520px)')`-driven IIFE makes long text expandable and is **responsive to resize**: `setup()` applies the collapse under 520px, `teardown()` removes the injected buttons and strips the inline styles above 520px (so resizing to desktop restores the full text), driven by the `matchMedia` `change` listener (`data-collapsible` guards prevent double setup). The About paragraph collapses to ~80px with a fade-out mask + "Read more"/"Read less"; each experience bullet list with >2 items collapses by animating the `<ul>`'s own `height` (single element = one layout recalc per frame), with the hidden `<li>`s fading via staggered `opacity`. Buttons are injected by JS only on small screens — no duplicate content.

**Responsive** — the `@media (max-width: 520px)` rule shows the hamburger and turns `.site-nav` into an animated, blurred dropdown, stacks banners, collapses the experience grid and date, and switches the skills grid and contact card to one column.

**Scrollbar** — a thin (4px) purple scrollbar via `::-webkit-scrollbar` (WebKit) and `scrollbar-color` (Firefox), with a transparent track. `.site-nav` hides its own (horizontal) scrollbar with `scrollbar-width: none` + `::-webkit-scrollbar { display: none }`.

## CV generator

`cv/generate_cv.py` builds `assets/CV_Mateusz_Filipczak.docx` — run it with `python3 cv/generate_cv.py` or the **`/cv`** slash command (`.claude/commands/cv.md`). It is **deterministic** (stdlib only — `html.parser`, `json`, `zipfile`, `zlib`; no clock/RNG/network), so the same inputs always produce a **byte-identical** `.docx` (zip entries are pinned to a fixed 1980-01-01 timestamp).

**Inputs:** shared sections (name, headline, summary, experience, skills, projects, contact) are **parsed from `index.html`** via a tiny `HTMLParser`→tree, keyed on the page's semantic classes (`.home-name`, `.home-tagline`, `.about-text`, `.exp-entry`/`.exp-title`/`.exp-date`/`.exp-company`/`.exp-promo`/`.exp-bullets`, nested `.exp-projects`/`.exp-project`/`.exp-project-name`, `.skill-category`/`h3`/`.skill-level`/`.skill-tag`, `.project-card`, `.contact-btn`/`.contact-row`). It raises a clear error if an expected anchor is missing — so a page refactor fails loudly instead of emitting an empty CV. **CV-only content** (Education, Certifications) and optional overrides live in **`cv/cv-extra.json`** (`education`, `certifications`, optional `contact`, `options.include_projects`, `options.section_order`). The DOCX styling mirrors the site (Segoe UI, purple `#8C6EDC` accents). The generator **owns** the output `.docx` — don't hand-edit it; change Education/Certs in `cv-extra.json` and regenerate. PDF was intentionally not implemented (export from Word, or add headless-Chrome/LibreOffice later).
